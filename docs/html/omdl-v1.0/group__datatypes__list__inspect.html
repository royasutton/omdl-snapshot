Conventions
  • Two-list parameters use v1 and v2.
  • The decimal-places parameter is always p.
  • almost_eq() applies a relative tolerance of eps (the library epsilon constant) by default; p overrides the number of significant decimal places used in the comparison.
  • compare() establishes a total order across OpenSCAD types: undef < number < boolean < string < list < range. Within each type, the natural ordering applies (numeric for numbers, lexicographic for strings, element-wise for lists).
  • list_get_value() performs a linear key search; for large maps prefer the map group functions which offer O(1) key lookup via search().

Test if all elements of two iterable values are sufficiently equal.

Parameters
v1<iterable> An iterable data type value 1.
v2<iterable> An iterable data type value 2.
p<number> The precision for numerical comparisons.
Returns
(1) <boolean> true when all elements of each iterable value are sufficiently equal and false otherwise.

The iterable values can be of mixed data types. All numerical comparisons are performed using the specified precision. All non-numeric comparisons test for equality. When both v1 and v2 are both numerical vectors, the function almost_eq_nv() provides a more efficient test.

Return values (rv) table.

rv order of values
-1 (v2 < v1)
0 (v2 == v1)
+1 (v2 > v1)

The following table summarizes how data type values are ordered.

order type s intra-type ordering
1 undef (singular)
2 number numerical comparison
3 boolean false < true
4 string lexical comparison
5 list compare (1) lengths, then (2) element-wise
6 range true compare sum of range elements
6 range false compare (1) lengths, then (2) element-wise

When comparing two lists of equal length, the comparison continues with successive elements until an ordering can be determined. Two lists are equal when all elements have been compared and no ordering has been determined.

Warning
The performance of element-wise comparisons degrades exponentially with list size and the sum of a range may exceeded the intermediate variable storage capacity for long ranges.

Return the value of an indexed list element with output defaults and list composition.

Parameters
l
input list.
i<integer> The input list element index.
dv<value> The default return value.
s<integer> The output list size.
de<value> The default element value for output list composition.
di<integer> The default output element index used when composing an output list from an input element that is not a list. Use di = -1 to assign the value to all elements.
Returns
For the input list element indexed as e = l[i]:
  • For e a list:
  1. When s == 0, returns (1) e[0], or (2a) dv if e[0] is undefined.
  2. When either de or s is undefined, returns (3a) e.
  3. When both de and s are defined with s > 0, returns (4) a list of size s in which any undefined elements of e are assigned de.
  • For e not a list:
  1. When s == 0, returns (2b) dv if e is undefined, and (3b) e otherwise.
  2. When either di or s is undefined, returns (2c) dv.
  3. When both di and s are defined with s > 0, returns (5) a list of size s with all elements assigned de if e is undefined, or (6) a list of size s with element [di] assigned e and all other elements assigned de.

Return conditions summary

no value description
1 e[0] element 0 of the indexed element (remove from list)
2 dv default value
3 e the indexed element
4 composed assign de to all undefined elements of e
5 composed assign de to all elements of e
6 composed assign e to element(s) di and de to all other element e

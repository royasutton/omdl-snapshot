54  \anchor parameter_naming_conventions
55  \par Parameter Naming Conventions
56 
57  The following naming conventions apply to all function and module
58  parameters across every group in this library. One name always
59  denotes one role; the same name is never reused for a different
60  purpose in a different group unless explicitly noted as group-local
61  below.
62 
63  \par Universal parameters
64 
65  These names carry the same meaning in every group.
66 
67  - \p v : the primary input value or iterable. Always the first
68  parameter unless the function is explicitly keyed on a
69  different primary (e.g. map functions key on \p m).
70  - \p cv : a comparison or reference value tested against \p v.
71  - \p d : a default or fallback value returned when the primary
72  input is undefined or a lookup fails.
73  - \p i : an element index (integer). Used exclusively for
74  positional access into an iterable; never repurposed
75  as an insert value or counter.
76  - \p n : an element count or length. Replaces the former uses
77  of \p c, \p l, and \p w in this role.
78  - \p mv : a match value; the element being searched for.
79  - \p mn : a match count; the maximum number of occurrences to
80  return. A value of \b 0 (or any value <= 0) means
81  return all occurrences.
82  - \p iv : an insert value. Used when a function inserts an
83  element into an iterable; kept distinct from \p i
84  (index) and \p mv (match value).
85  - \p i1 / \p i2 : the start and end index of a search window,
86  respectively.
87  - \p dp : decimal places; the number of significant fractional
88  digits used in a numeric comparison or rounding
89  operation.
90 
91  \par Bounds parameters
92 
93  - \p v_min : the lower bound of a numeric range (inclusive). Replaces
94  the former \p l and \p u (when used as lower) in scalar
95  and list operations.
96  - \p v_max : the upper bound of a numeric range (inclusive). Replaces
97  the former \p u when used as upper bound.
98 
99  \par Internal recursion parameters
100 
101  Parameters whose names begin with an underscore are internal
102  recursion variables. They must \b never be initialized by the
103  caller; doing so produces undefined results. All such parameters
104  are tagged \em internal in their \p \\param entry.
105 
106  - \p _acc : a running accumulator (replaces \p bv, \p e, bare \p c
107  used as counters).
108  - \p _bv : the current bit-value accumulator in binary operations
109  (replaces the public \p bv).
110  - \p _bm : the current bit-mask accumulator in binary operations
111  (replaces the public \p bm).
112  - \p _i : an internal index counter (replaces bare \p c used as a
113  traversal counter in all_lists(), all_strings(),
114  all_numbers(), and all_len()).
115 
116  \par Boolean flag parameters
117 
118  Boolean flags always carry a descriptive name that makes their
119  purpose readable at the call site without consulting documentation.
120 
121  - \p use_search : selects the element-matching method: \b true uses
122  search(), \b false uses find(). Replaces
123  \p s in functions where \p s was used solely as a
124  method selector.
125  - \p ri_is_value : indicates whether a row identifier argument is a
126  literal value rather than an index string. Replaces
127  \p vri in table operations.
128  - \p ci_is_value : indicates whether a column identifier argument is a
129  literal value rather than an index string. Replaces
130  \p vci in table operations.
131  - \p ignore : suppress assertion on missing keys in map
132  operations. Kept as-is; already descriptive.
133 
134  \par Group-local parameters
135 
136  The following names have a single well-defined role within their
137  group. They are \b not repurposed outside that group.
138 
139  - \p b : a bit position (binary group only).
140  - \p w : a bit width (binary group only).
141  - \p s : a shift amount (binary group only).
142  - \p p : a point coordinate, as \b list-2d or \b list-3d
143  (euclidean group only). The two-point form uses
144  \p p1 and \p p2.
145  - \p l : a directed line, represented as
146  \b [p_initial, p_terminal] (euclidean group only).
147  Not used as list-length or lower-bound anywhere.
148  - \p n : a plane normal vector (euclidean group only).
149  Context distinguishes this from the universal \p n
150  (count); the euclidean group has no count
151  parameters.
152  - \p o : an origin point for spatial transformations
153  (euclidean group only).
154  - \p a / \p av : a rotation angle or rotation-axis vector
155  (euclidean group only).
156  - \p x / \p y : Cartesian coordinates for interpolation
157  (euclidean group only).
158  - \p m : a map, i.e. a list of \b [key, value] pairs (map
159  group only). \p m2 denotes a second map in
160  two-map operations.
161  - \p k : a map key of any valid value type (map group only).
162  - \p u : an update map whose entries are merged into \p m
163  (map group only).
164  - \p r : the row data matrix of a table (table group only).
165  - \p c : the column-identifier matrix of a table (table
166  group only). Not used as match-count outside this
167  group.
168  - \p ri / \p ci : a row or column identifier string used to look up
169  a table entry (table group only).
170  - \p si : the sub-element index used as the sort key in
171  sort_q2() (list group only).
172  - \p col : the column selector index passed to select_e()
173  and related functions (list group only).
